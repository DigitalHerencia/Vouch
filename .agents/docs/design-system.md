# Vouch Design System

Source of truth date: 2026-04-30

Vouch UI should feel serious, precise, neutral, professional, secure, fast, and clear.

## Language

Use:

- merchant/provider
- customer/client
- commitment-backed payment
- protected amount
- Vouch service fee
- processing fee offset
- customer total
- confirm presence
- release funds
- refund

Avoid:

- escrow
- marketplace
- book now
- dispute
- claim
- judge
- rating
- review
- message

## Required Payment Display

Before the customer/client pays, show:

- protected amount
- merchant receives
- Vouch service fee
- processing fee offset
- customer total

## Vouch Screen Checklist

Every Vouch screen should answer:

- current status
- protected amount and customer total where relevant
- merchant/provider and customer/client roles
- required next action
- deadline
- what happens if no action occurs
- whether each participant has acted
- whether the outcome is final
